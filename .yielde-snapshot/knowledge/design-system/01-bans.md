# Banned Patterns

A short list. Banned not because they're generic, but because their execution is almost always thoughtless.

| Pattern | Why |
|---|---|
| Stock photo hero with dark gradient overlay + centred white headline | The gradient exists to make any photo work without thinking about it. That is avoidance of design, not design. |
| Symmetric 3-column icon grid for "features" or "services" every time | Identical columns flatten hierarchy. Layout should reflect content priority — not force everything into an equal mould. |
| `opacity: 0 → 1` + `translateY(20px) → 0` as the scroll animation | The default scroll reveal in every UI library. Using it without thought is building a template. |
| Entrance animation on every element on every scroll event | When everything moves, motion communicates nothing. Reserve it for elements that earn it. |
| "Our Services / Our Values / Our Team" as the page's three acts | This is the structure of a company no one has met yet. Real clients have a real story — find it and build from that. |
| Floating blobs, mesh gradients, radial halos behind headlines | See [Motion Bans](@knowledge/design-system/06-motion.md#73-banned). |

| Purple | See [Semantic Separation](@knowledge/design-system/02-colour-rules.md#32-semantic-separation). |
<!-- YIELDE:END -->

> These are bans on **thoughtless use**, not on the pattern itself.
> If a client requests one, document it in their brief and execute it with enough craft that it earns its place.
