# Pre-Build Ritual

Complete this before writing code or placing any component. Not optional.

**Step 1 — Survey the space.**
Open 3–5 real sites in this client's industry. Note what they all share: layout patterns, colour conventions, how the hero is structured, how pricing is presented, what the nav looks like.

**Step 2 — Name the pattern.**
Write one sentence: *"Sites in [this industry] all [do X]."* If you can't name it, look harder.

**Step 3 — Make a decision.**
For each shared pattern — is it there because it's genuinely the best solution, or because everyone copied everyone?
- **Best solution** → do it, but execute it with more craft than anyone else in that space.
- **Just inherited** → replace it with something better, or cut it.

**Step 4 — Log it.**
One line in the client's brief (`Forms/[client].md`): *"Industry default: [X]. Our answer: [Y]."* This creates a paper trail and forces the decision to be conscious.

> The point is not to be different. It is to be intentional.
> A centred hero with a clean headline is not a problem.
> Placing it there without asking why is.

*Execution of this ritual — when to run it, how to prompt the user, and how to log it — is coordinated by the `/plan-build` command.*
